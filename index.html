Inspired Meals 4 You

Inspired Meals 4 You is the first group project I worked on in UNC-CH's Full Stack Bootcamp. We created a webpage that took 3 parameter inputs from the user: daily calorie goal, foods to exclude and meal preference to create a weekly meal plan for the user. Each day has three meals, breakfast, lunch and dinner, and the prep/cook time, serving sizes, instructions, ingredients and title are given for each recipe. The user can also save the recipes they enjoy to a favorites page.

The APIs used are: the spoonacular Nutrition, Recipe and Food API and Quotes Inspirational Quotes Motivational Quotes API.

My main roles were to create the opening page' and favorites pages' html and js files. I also helped with the js on the main page that involved using the local storage to connect to the spoonacular Nutrition, Recipe and Food API to use the 3 parameteres collected on the opening page to get our weekly recipes and the name and webpages for our favorites list page.

Work Day Scheduler

Work Day Scheduler was a homework assignment with UNC Chapel Hill Full Stack Boot Camp. I created a work day calendar that was separated by hour, with an events input area and a button to save the input. There are nine hours on the calendar, 9am through 5pm, and the input for each hour will not be removed upon refreshing the page. The only way the event can change is if the user decides to change the input information. Depending on the hour of the day the event input background color changes based on the current time.

I created the page using bootstrap themes with additional css. Moment.js is used to get the current time in military time. By comparing the individual hour ids, which were based on military time, to the current time in a for loop with if, else if, and else statements, I was able to append new background color classes to the specific Ids. The colors are grey for past, red for present and green for future events.

Local Storage was used to store the event input in a key value pair. Each key is named after the specific hours Id. The event is then appended to the correct event using a for loop, to get the correct values from local storage and place them under the correct input id that matched the key name.

Weather Dashboard

Weather Dashboard was a homework assignment with UNC Chapel Hill Full Stack Boot Camp. The user inputs a city and they are provided with the current weather information as well as a five-day forecast. Also, any cities that are searched appear as buttons with their names on it below the main search. Now these previous searches become their own city-specific search buttons.

I used the Open Weather Map API to gather the information for this page. First, I used the geolocate of the city from the input to get the latitude and longitude of the city’s location. Then these two parameters are plugged into another Open Weather Map API that uses latitude and longitude to provide forecast information.

With the response from these second API, I could gather the information needed for the current day and the next 5 days: temperature, wind speed, humidity and an icon that represented the current type of weather. The current day also included the UV index and in JavaScript a for loop is used to change the UV background color based on the number. Moment.js was used to give the current and the five day forecast dates.

Bookworm CLM

Bookworm CLM (Classroom Library Management) is the third group project I worked on in UNC-CH's Full Stack Bootcamp. We used a MERN stack to create a webpage that allows teachers to keep track of their personal classroom libraries. Only teachers are the users and they have a saved list of their students and books. There is a search engine, using google books, that allows the teacher to search for books by Title or ISBN. Once they find the book they are looking for, they can add the number of copies before it is saved to their library. There is also a log that keeps track of which books are checked out and by which students, and the date they are due back.

The app was created using React and we used the Google Books API to generate the information about the books, displaying the title, author, description and an image of the cover. MongoDB, with GraphQL and Apollo, are used to store the teacher information. Currently their students, books and log are embedded as array's within the teacher model. The website uses JWT Authentication and Bcrypt to ensure a secure login.

I worked mainly on the backend and database: creating models, the typeDefs and resolvers schemas. I also helped connected the backend to the frontend writing the mutations and queries as well as the code in the components to call on these mutations and queries.

Workout Tracker

Workout Tracker was a homework assignment with UNC Chapel Hill Full Stack Boot Camp. We needed to create a server, API and HTML routes and models to make this a fully functioning website.

In the server.js Express is needed to create REST APIs. In this file the routes for the api.js and view.js files are created. Also mongoose.connect is required for connecting to Atlas MongoDB when using Heroku or Robo 3T on a local host.

The models needed were Workout.js and index.js to export. The Workout model only requires "type", "name" and duration" because between the two types of workouts the name and the duration are the only two shared properties. The other properties are displayed depending on the workout type chosen.

The REST API's need to include GET to display daily workouts, POST to create a new workout, PUT to update the current workout. Also, a specific GET to display the workouts for the last 7 days.

Budget Tracker

Budget Tracker was a homework assignment with UNC Chapel Hill Full Stack Boot Camp. The goal was to make this application functional offline and to make it a PWA. In order to do this i created a manifest.webmanifest file, a db.js file for the indexedDB and a service-worker.js file for both the manifest and the database.

The Manifest file is needed so the PWA is discoverable. The file is an object that provides the PWA parameters, how it will look for the user as an app. I chose to make it stand alone and have portrait orientation so it had the appearance of a freestanding app.

There are two main functions in the indexedDB. The first "saveRecord" will hold the data entered for the budget while the app is offline in the cache. The second is "checkDatabase" which checks everytime the app is open whether it has access to the internet and if it does it sends the cached data to MongoDB and deletes the cache.

The Service Worker is used to retrieve information from the cache, for certain network requests and to cache information or data. The service worker is needed by the PWA because is grabs the files to cache, the static cache and the data to cache. The PWA needs this information to work online and offline.
